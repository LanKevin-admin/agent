# 分发包制作与使用指南

## 🎯 目标

制作一个**单EXE文件**，分享给别人后，别人只需要：
1. 双击EXE启动
2. 配置DeepSeek API Key
3. 开始使用

---

## 📦 重新打包EXE

### 前置条件

确保前端已构建：
```bash
cd web
npm run build
cd ..
```

### 打包命令

```bash
pyinstaller --clean feishu_rpa_monitor.spec
```

### 输出位置

```
dist/
└── 飞书RPA监控系统.exe  （约130MB）
```

---

## 📤 制作分发包

### 方案1：纯净版（推荐给新用户）

**只需要分发1个文件：**
```
飞书RPA监控系统.exe
```

**用户首次启动时：**
1. 程序自动创建必要的文件夹（`data/`、`reports/`）
2. 浏览器自动打开配置页面
3. 填写DeepSeek API Key
4. 重启程序生效

**优点：**
- 最简单，只需一个文件
- 用户通过Web界面配置，友好

---

### 方案2：带说明版（推荐）

**分发包包含：**
```
飞书RPA监控系统/
├── 飞书RPA监控系统.exe       # 主程序
├── README.md                  # 快速开始指南
├── 使用说明.md                # 详细说明
└── .env.example               # 配置文件模板（可选）
```

**制作步骤：**
```bash
# 1. 创建分发文件夹
mkdir 飞书RPA监控系统分发包
cd 飞书RPA监控系统分发包

# 2. 复制EXE
copy ..\dist\飞书RPA监控系统.exe .

# 3. 复制说明文件（这些文件已经打包在EXE中，但也可以单独提供）
copy ..\README.md .
copy ..\使用说明.md .
copy ..\.env.example .

# 4. 打包成ZIP
# 右键 -> 发送到 -> 压缩文件夹
```

**优点：**
- 用户可以先看README了解如何使用
- 高级用户可以直接修改.env文件

---

### 方案3：预配置版（内部使用）

**适用场景：**
- 给公司内部员工使用
- API Key等配置已经统一

**分发包包含：**
```
飞书RPA监控系统/
├── 飞书RPA监控系统.exe
├── .env                        # ✅ 预配置好的环境变量
├── README.md
└── 使用说明.md
```

**制作步骤：**
```bash
# 1. 创建.env文件，填入统一配置
# 2. 把.env和EXE一起打包
```

**优点：**
- 用户开箱即用
- 无需配置

**缺点：**
- API Key暴露在文件中（需要注意安全）

## 🚀 用户使用流程

### 方案1和2的使用流程

1. **解压/双击EXE**
2. **等待启动**（控制台显示启动信息）
3. **浏览器自动打开**（如未打开，手动访问 http://127.0.0.1:8888）
4. **首次配置**：
   - 点击左侧菜单"系统配置"
   - 填写DeepSeek API Key、API Base、模型名
   - 点击"保存配置"
5. **重启程序**（关闭控制台窗口，重新双击EXE）
6. **开始使用**：
   - 访问"智能对话"页面
   - 输入：`你好，分析一下今天的日志`
   - 查看AI回复

## 📞 支持与反馈

### 常见问题处理

**Q: 用户说启动失败**
- 让用户截图控制台错误信息
- 检查是否有杀毒软件拦截
- 确认Windows版本（建议Win10及以上）

**Q: 用户说配置不生效**
- 确认是否重启了程序
- 检查.env文件是否在EXE同目录
- 检查配置项格式是否正确

**Q: 用户说端口被占用**
- 检查8888端口是否被其他程序占用
- 可以修改web_server.py中的默认端口
