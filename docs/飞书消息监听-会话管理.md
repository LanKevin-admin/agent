# 飞书消息监听 - 会话管理功能

## 🎯 功能概述

机器人现在支持**智能会话管理**，可以识别并处理两种场景的消息：

### 场景1：先@，后续不@（会话保持）

```
用户：@机器人 帮我分析今天的日志
机器人：（分析结果...）

用户：再看看昨天的    👈 没有@，但机器人仍然会回复
机器人：（昨天的分析...）

用户：对比一下两天的差异
机器人：（对比分析...）
```

**特点：**
- ✅ 首次必须@机器人
- ✅ 后续5分钟内不需要@
- ✅ 自动维护会话上下文
- ✅ 超过5分钟需要重新@

---

### 场景2：连续快速@（独立处理）

```
用户：@机器人 第一个问题
用户：@机器人 第二个问题
用户：@机器人 第三个问题
```

**行为：**
- ✅ 每条消息独立处理
- ✅ 每条都会收到回复
- ✅ 按照时间顺序处理

---

## 🏗️ 技术实现

### 1. 会话管理器

```python
# 数据结构
user_sessions = {
    "user_id_123": {
        "last_mention_time": 1704067200,  # 最后@的时间戳
        "context": []                      # 预留：对话上下文
    }
}

# 配置
session_timeout = 300  # 5分钟会话超时
```

### 2. 消息识别逻辑

```python
# 步骤1：检查是否直接@机器人
if is_mention_bot:
    处理消息 + 创建/更新会话

# 步骤2：检查是否同一用户的后续消息
elif user_id in active_sessions:
    if 时间差 <= 5分钟:
        处理后续消息
    else:
        会话已过期，忽略
```

### 3. 防重复机制

```python
# 使用消息ID去重
processed_message_ids = set()

# 每条消息只处理一次
if message_id in processed_message_ids:
    跳过
```

## ⚙️ 配置参数

### 会话超时时间

```python
# 在 message_monitor.py 中修改
session_timeout = 300  # 秒，默认5分钟

# 建议值：
# - 3分钟 (180)：快速过期，节省资源
# - 5分钟 (300)：平衡体验和资源
# - 10分钟 (600)：更长的会话保持
```

### 消息检查间隔

```python
# 在 web_server.py 中修改
start_message_monitor(agent, interval=10)  # 秒

# 建议值：
# - 5秒：更快响应，但API调用更频繁
# - 10秒：平衡响应速度和API频率
# - 30秒：节省API调用，但响应较慢
```

## 📝 注意事项

### 1. 消息时间窗口

- 只检查**最近1分钟**的消息
- 如果消息超过1分钟前发送，检测不到
- 配合10秒检查间隔，确保不漏消息

### 2. 会话过期

- 超过5分钟无交互，会话自动过期
- 过期后需要重新@机器人
- 会话过期不会影响已处理的消息

### 3. 并发处理

- 多个用户可以同时有活跃会话
- 每个用户的会话独立管理
- 不同用户之间互不影响

### 4. 性能考虑

- 已处理消息ID最多记录1000条
- 活跃会话数没有限制
- 会话过期自动清理，不会内存泄漏
