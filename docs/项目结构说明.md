# 飞书RPA监控系统 - 项目结构说明

## 📁 整体结构

```
feishu_rpa_monitor/
├── agent/                  # AI Agent核心模块
├── config/                 # 全局配置
├── database/               # 数据库模型和操作
├── feishu/                 # 飞书API集成
├── notify/                 # 通知模块（邮件、企业微信）
├── prompts/                # AI提示词
├── report/                 # 报告生成
├── scheduler/              # 定时任务调度
├── skills/                 # Skills架构（9个技能模块）
├── utils/                  # 工具函数
├── web/                    # Vue 3前端项目
├── web_server.py           # FastAPI后端服务器
├── requirements.txt        # Python依赖
├── feishu_rpa_monitor.spec # PyInstaller打包配置
├── README.md               # 快速开始指南
└── 使用说明.md             # 详细使用文档
```

---

## 🎯 后端结构详解

### 1. agent/ - AI Agent核心

```
agent/
├── skill_based_agent.py        # 主Agent（基于Skills架构）
├── tools.py                    # 工具函数定义（给AI调用）
├── analyzer.py                 # 日志分析器
├── log_parser.py               # 日志解析器
├── file_validator.py           # 文件校验器
├── config_tools.py             # 配置管理工具
├── db_tools.py                 # 数据库操作工具
└── scheduled_task_tools.py     # 定时任务工具
```

**核心类：**
- `SkillBasedAgent` - 主Agent，整合所有Skills，调用OpenAI API

---

### 2. skills/ - Skills架构

```
skills/
├── skill_manager.py            # Skills管理器
├── base_skill.py               # Skills基类
├── feishu_skill.py             # 飞书消息技能
├── file_validation_skill.py    # 文件校验技能
├── database_skill.py           # 数据库查询技能
├── notification_skill.py       # 通知发送技能
├── report_skill.py             # 报告生成技能
├── config_skill.py             # 配置管理技能
├── scheduled_task_skill.py     # 定时任务技能
├── task_planning_skill.py      # 任务规划技能
└── turing_test_skill.py        # 图灵测试技能
```

**设计模式：** 参考OpenClaw的Skills架构
- 每个Skill独立封装业务逻辑
- 通过`SkillManager`统一管理和路由
- 工具函数通过装饰器自动注册

---

### 3. config/ - 配置管理

```
config/
├── settings.py                 # 全局配置类
└── .env                        # 环境变量（敏感信息）
```

**配置类：**
- `FeishuConfig` - 飞书应用配置
- `EmailConfig` - 邮件配置
- `AIConfig` - AI模型配置
- `SchedulerConfig` - 定时任务配置
- `WeComConfig` - 企业微信配置
- `ReportConfig` - 报告输出配置

---

### 4. database/ - 数据库

```
database/
├── models.py                   # SQLAlchemy模型定义
└── operations.py               # 数据库操作封装
```

**数据表：**
- `conversations` - 对话历史
- `scheduled_tasks` - 定时任务
- `task_executions` - 任务执行记录
- `analysis_templates` - 分析模板

**ORM：** SQLAlchemy 2.0 + SQLite

---

### 5. feishu/ - 飞书集成

```
feishu/
├── client.py                   # 飞书API客户端
├── message_fetcher.py          # 消息拉取
├── message_sender.py           # 消息发送
└── log_filter.py               # 日志过滤器
```

**功能：**
- 获取tenant_access_token
- 拉取群聊消息
- 发送文本消息
- 过滤RPA日志

---

### 6. notify/ - 通知模块

```
notify/
├── email_sender.py             # 邮件发送（支持附件）
└── wecom_sender.py             # 企业微信机器人
```

**支持：**
- SMTP邮件发送
- 邮件附件（报告TXT）
- 企业微信Webhook

---

### 7. report/ - 报告生成

```
report/
├── executive_report.py         # 领导级汇报生成器
└── generator.py                # 通用报告生成器
```

**功能：**
- 生成文本摘要
- 生成详细报告文件
- 统计失败项和趋势

---

### 8. scheduler/ - 定时任务

```
scheduler/
└── task_scheduler.py           # APScheduler调度器
```

**功能：**
- 启动时加载数据库中的任务
- CronTrigger定时执行
- 执行记录保存到数据库

---

### 9. prompts/ - AI提示词

```
prompts/
└── system_prompt.md            # 系统提示词（人格设定）
```

**人格：** 兰工（5年运维经验，专业友好）

---

### 10. web_server.py - FastAPI后端

**主要功能：**
- `/api/health` - 健康检查
- `/api/chat` - 智能对话
- `/api/analysis` - 数据分析
- `/api/config` - 配置管理
- `/api/tasks` - 定时任务管理
- `/api/templates` - 模板管理
- `/api/reports` - 报告下载
- 静态文件服务（Vue前端）

**端口：** 8888

## 🎨 前端结构详解

### web/ - Vue 3项目

```
web/
├── src/
│   ├── main.js                # 入口文件
│   ├── App.vue                # 根组件
│   ├── api.js                 # API调用封装
│   ├── components/            # 可复用组件
│   │   └── SensitiveInput.vue # 敏感信息输入组件
│   └── views/                 # 页面组件
│       ├── Dashboard.vue      # 首页仪表盘
│       ├── Chat.vue           # 智能对话
│       ├── Analysis.vue       # 数据分析
│       ├── Reports.vue        # 历史报告
│       ├── Tasks.vue          # 定时任务
│       ├── Templates.vue      # 分析模板
│       └── Config.vue         # 系统配置
├── index.html                 # HTML模板
├── package.json               # npm依赖
├── vite.config.js             # Vite配置
```

### 技术栈

- **Vue 3** - Composition API + `<script setup>`
- **Vue Router 4** - Hash模式路由
- **Element Plus 2.x** - UI组件库
- **axios** - HTTP请求
- **Vite 5** - 构建工具

### 主要页面功能

| 页面 | 路由 | 功能 |
|------|------|------|
| Dashboard | `/` | 系统状态、最近分析、快速操作 |
| Chat | `/chat` | AI对话、历史记录、实时交互 |
| Analysis | `/analysis` | 手动分析、日期范围选择 |
| Reports | `/reports` | 历史报告列表、下载 |
| Tasks | `/tasks` | 定时任务管理、执行历史 |

## 🗄️ 数据库设计

### SQLite数据库（`data/rpa_monitor.db`）

**表结构：**

```sql
-- 对话历史
conversations (
    id INTEGER PRIMARY KEY,
    role TEXT,
    content TEXT,
    timestamp DATETIME
)

-- 定时任务
scheduled_tasks (
    id INTEGER PRIMARY KEY,
    name TEXT,
    cron_expression TEXT,
    query_text TEXT,
    enabled BOOLEAN,
    created_at DATETIME,
    updated_at DATETIME
)

-- 任务执行记录
task_executions (
    id INTEGER PRIMARY KEY,
    task_id INTEGER,
    status TEXT,
    result TEXT,
    executed_at DATETIME,
    duration REAL
)

-- 分析模板
analysis_templates (
    id INTEGER PRIMARY KEY,
    name TEXT,
    description TEXT,
    config JSON,
    created_at DATETIME,
    updated_at DATETIME
)
```

## 📦 打包相关

### feishu_rpa_monitor.spec

**PyInstaller配置文件**

**打包内容：**
- Python运行时 + 所有依赖
- web/dist（Vue前端静态文件）
- prompts/（系统提示词）
- .env（默认配置）

**输出：** `dist/飞书RPA监控系统.exe`（约41MB）

**运行时文件结构：**
```
飞书RPA监控系统.exe同目录/
├── .env                        # 外部配置（可修改）
├── data/                       # 数据目录
│   └── rpa_monitor.db
└── reports/                    # 报告输出
    └── *.txt
```

## 🔧 开发与构建

### 开发模式

**后端：**
```bash
python web_server.py
```
访问：http://127.0.0.1:8888

**前端：**
```bash
cd web
npm run dev
```
访问：http://localhost:5173

### 生产构建

**前端构建：**
```bash
cd web
npm run build    # 输出到 web/dist/
```

**打包EXE：**
```bash
pyinstaller --clean feishu_rpa_monitor.spec
```

## 📝 配置文件

### .env 环境变量

所有敏感配置通过`.env`文件管理：
- AI API配置（DeepSeek）
- 飞书应用凭证

- 企业微信Webhook

### 配置优先级

1. **外部.env**（EXE同目录）- 用户可修改
2. **打包内.env** - 默认值
3. **代码默认值**

1. **Skills架构** - 模块化、可扩展

5. **配置外置** - 用户友好
6. **数据持久化** - SQLite轻量化
7. **定时调度** - APScheduler后台执行
